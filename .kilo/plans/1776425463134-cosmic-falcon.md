# Plan: Interactive Demo with GitHub Pages

**Branch:** `docs/demo-sample`  
**Type:** Documentation Feature (non-code change to library)  
**Objective:** Create an interactive web-based demo of GeoPathFinder accessible via GitHub Pages

## Overview

Build a Vite-powered interactive demo where users can:

- Draw start point, target points, and polygonal obstacles (via Leaflet + GeoMan)
- Configure search parameters (distanceMax, algorithm)
- Load predefined scenarios (3-5 JSON scenarios)
- Visualize pathfinding results on a map

Deployment: Automated via GitHub Actions workflow → `gh-pages` branch → GitHub Pages

## Architecture

```
demo/                          # Source files (committed to main)
├── public/
│   ├── index.html             # Main entry point
│   └── scenarios.json         # Predefined scenarios (3-5 examples)
├── src/
│   ├── main.ts                # Entry point
│   ├── app.ts                 # Main app logic
│   ├── ui/
│   │   ├── controls.ts        # UI controls (algorithm, distance, load scenario)
│   │   └── styles.css         # Styles
│   ├── map/
│   │   ├── mapManager.ts      # Leaflet + GeoMan integration
│   │   └── visualization.ts   # Draw path results
│   └── worker/
│       ├── search.ts          # GeoPathFinder search handler
│       └── loader.ts          # Load scenarios
├── vite.config.ts             # Vite configuration
├── tsconfig.json              # TypeScript config

.github/workflows/
└── deploy-demo.yml            # GitHub Actions: build Vite → push to gh-pages

```

## Implementation Tasks

### Phase 1: Setup & Configuration

1. **Create demo directory structure**
   - Initialize `demo/` folder
   - Create `public/`, `src/`, `src/ui/`, `src/map/`, `src/worker/` subdirectories
   - Create `vite.config.ts`, `tsconfig.json`, `package.json`

2. **Initialize demo dependencies**
   - Install: `geopathfinder`, `leaflet`, `@geoman-io/geoman-leaflet`, `vite`, `typescript`
   - Create lock file (yarn)
   - Verify installation works

3. **Create Vite configuration**
   - Configure build output → `docs/` directory
   - Configure TypeScript support
   - Test: `vite` (dev server) and `vite build` (production build)

### Phase 2: Map & UI

4. **Implement Leaflet + GeoMan integration**
   - Initialize map in `map/mapManager.ts`
   - Enable GeoMan drawing tools (point, polygon)
   - Implement drawing event listeners (capture start, targets, obstacles)
   - Visualization in `map/visualization.ts` for results

5. **Build UI controls**
   - Input: `distanceMax` (meters)
   - Dropdown: `shortestPathAlgorithm` ('a\*' | 'dijkstra')
   - Button: "Search"
   - Button: "Clear"
   - Dropdown: "Load Scenario"
   - Status display (loading, success, error)

### Phase 3: Scenarios & Search

6. **Create predefined scenarios**
   - Define 3-5 JSON scenarios in `public/scenarios.json`
   - Re-use existing scenarios from `profile/` or create new ones
   - Include: start, targets, obstacles, distanceMax for each

7. **Implement search handler**
   - Load `geopathfinder` from npm
   - Create VisibilityGraph instance with user input
   - Call `graph.search(distanceMax, options)`
   - Handle results & errors
   - Visualize path on map

8. **Implement scenario loader**
   - Fetch `scenarios.json`
   - Parse and populate dropdown
   - Load scenario → pre-fill map with geometry

### Phase 4: Documentation & Deployment

9. **Create demo README**
   - Usage instructions
   - How to install deps locally: `cd demo && yarn install`
   - How to start dev server: `cd demo && yarn dev`
   - How to build: `cd demo && yarn build`
   - Scenario format documentation

10. **Create GitHub Actions workflow**
    - Trigger: push to `main` branch
    - Steps:
      1. Checkout repo
      2. Setup Node.js
      3. Install demo deps: `cd demo && yarn install`
      4. Build demo: `cd demo && yarn build`
      5. Deploy to `gh-pages`:
         - Use `actions/upload-pages-artifact` (upload `docs/`)
         - Use `actions/deploy-pages` (deploy to GitHub Pages)
    - Configure GitHub Pages settings to use `gh-pages` branch

11. **Test deployment**
    - Verify GitHub Pages is configured to use `gh-pages` branch
    - Check demo is accessible at `https://razielwar.github.io/geopathfinder/`
    - Test interactivity in browser

### Phase 5: Integration & Cleanup

12. **Merge `docs/demo-sample` → `main`**
    - Ensure all tasks completed
    - Create PR with summary
    - Code review & merge

13. **Verify everything works**
    - GitHub Actions workflow runs successfully
    - Demo deploys to GitHub Pages
    - All features functional (draw, search, scenarios)

## Key Technical Details

### Scenarios JSON Format

```json
{
  "scenarios": [
    {
      "name": "Simple Route",
      "distanceMax": 2000000,
      "start": { "type": "Point", "coordinates": [0, 0] },
      "targets": [{ "type": "Point", "coordinates": [10, 10] }],
      "obstacles": [
        {
          "type": "Polygon",
          "coordinates": [
            [
              [5, 5],
              [6, 5],
              [6, 6],
              [5, 6],
              [5, 5]
            ]
          ]
        }
      ]
    }
  ]
}
```

### GitHub Actions Workflow

- Use `actions/setup-node@v4` for Node.js setup
- Use `actions/upload-pages-artifact@v3` to upload build
- Use `actions/deploy-pages@v4` to deploy
- Requires GitHub Pages environment setup (automatic or manual)

### Dependencies

- `geopathfinder`: Latest from npm
- `leaflet`: ^1.9.x
- `@geoman-io/geoman-leaflet`: ^2.14.x
- `vite`: ^5.x
- `typescript`: ^5.x

## Questions & Decisions

**Resolved:**

- ✅ GitHub Pages: Use `gh-pages` branch (not `docs/` folder) to avoid main branch protections
- ✅ Build output: Vite → `docs/` directory
- ✅ Automation: GitHub Actions workflow (triggered on push to `main`)
- ✅ Scenarios: JSON format, 3-5 examples
- ✅ geoPathFinder version: Public npm package

**Potential decisions for user:**

- Should demo be responsive/mobile-friendly? (Recommend: yes, but MVP can be desktop-first)
- Should we include a "full-screen map" option? (Recommend: yes, simple toggle)
- Default distanceMax value? (Recommend: 5,000,000 meters = 5000 km)
